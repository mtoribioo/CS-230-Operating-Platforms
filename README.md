# CS-230-Operating-Platforms

## Briefly summarize The Gaming Room client and their software requirements.

The client for this project was The Gaming Room, a company that wanted to expand their Android game, *Draw It or Lose It*, into a web-based application that could support multiple operating systems and mobile devices. The application needed to support multiple games, teams, and players while preventing duplicate game names. It also needed to be secure, scalable, and capable of supporting many users at the same time.

## What did you do particularly well in developing this documentation?

I think I did a good job organizing the software design document and explaining my design decisions. I clearly evaluated different operating platforms, recommended Linux as the best server platform, and explained how memory management, storage, networking, and security would support the application's requirements. I also made sure the document was easy to follow for both the client and the development team.

## What about the process of working through a design document did you find helpful when developing the code?

Working through the design document first made the coding process much easier. Creating the UML diagram and identifying the relationships between classes helped me understand how the application should be structured before writing any code. Having a clear design reduced confusion and made it easier to implement features like inheritance, the Singleton pattern, and collections.

## If you could choose one part of your work on these documents to revise, what would you pick? How would you improve it?

If I could revise one part of the document, I would expand the evaluation of the different operating platforms by providing additional comparisons of cloud hosting options and estimated deployment costs. This would provide the client with even more information when choosing the best platform for future growth.

## How did you interpret the user's needs and implement them into your software design? Why is it so important to consider the user's needs when designing?

I focused on the client's need for a scalable, secure, and cross-platform application. Those requirements influenced my recommendation to use a web-based architecture, Linux servers, centralized storage, and role-based security. Considering the user's needs is important because the software should solve the client's problem while remaining reliable, maintainable, and easy for users to access.

## How did you approach designing software? What techniques or strategies would you use in the future to analyze and design a similar software application?

I approached the design by first understanding the project requirements and then breaking the application into smaller components. I used object-oriented design principles, UML diagrams, and software design patterns to organize the system before writing code. In future projects, I would continue using these strategies while also creating prototypes earlier and gathering user feedback throughout the design process to improve the final product.
